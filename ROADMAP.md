# rustMH (ex-goMH) - Миграция на Rust 🦀

## Цели проекта
- Заменить консольный интерфейс (TUI) на полноценный touch-friendly GUI (`egui`).
- Оставить приложение портативным единым `.exe` файлом.
- Перевести системную логику Windows, настройку кассового ПО (iiko) и ККТ на Rust (`tokio` + WinAPI).

---

## Этап 1: Фундамент и Архитектура 🏗️
- [ ] Инициализировать проект (Cargo.toml).
- [ ] Настроить Release-профиль (минимизация размера `.exe`, LTO, strip).
- [ ] Настроить `build.rs` для включения Windows Manifest (запрос UAC / прав Администратора).
- [ ] Реализовать архитектуру mpsc-каналов: отделение синхронного UI (`egui`) от асинхронного Backend (`tokio`).
- [ ] Настроить базовое логирование (`tracing` / `tracing-subscriber`), которое пишет и в файл, и в UI.

## Этап 2: Dashboard и UI (Active Monitoring) 📊
- [ ] Создать фоновый воркер сбора метрик Windows через `sysinfo`.
- [ ] Разработать базовый Layout в `egui` (Боковое меню навигации, верхняя панель статуса).
- [ ] Адаптировать тему и размеры виджетов под сенсорные экраны (Touch-friendly: `pixels_per_point`).
- [ ] Добавить виджеты мониторинга: загрузка CPU, RAM, активность дисков (индикаторы чтения/записи) с использованием `egui_plot`.

## Этап 3: Базовые утилиты и Задачи (Task Queue) ⚙️
- [ ] Реализовать `TaskQueue` (Очередь задач) на стороне Backend'а.
- [ ] Перенести логику скачивания файлов (`reqwest`) с передачей прогресс-бара в UI.
- [ ] Перенести логику работы с архивами (`zip`, `7z` via CLI).
- [ ] Сделать красивый виджет в UI для отображения текущих задач и прогресс-баров загрузки/установки.
- [ ] Настроить CLI-парсер (`clap`). Если передан флаг `automation`, приложение должно работать в headless-режиме (без запуска `egui`).

## Этап 4: Системные функции и Службы Windows 🪟
- [ ] Реализовать управление Windows-службами (старт, стоп, статус) через `windows-service` или WinAPI.
- [ ] Перенести модуль работы с Реестром Windows (`winreg`).
- [ ] Перенести модуль сбора логов (Log Collector).

## Этап 5: Бизнес-логика iiko/Syrve 🍔
- [ ] Перенести менеджер дистрибутивов (Asset Manager) — скачивание нужных версий iiko.
- [ ] Реализовать генерацию и патчинг XML конфигураций (`config.xml`) с помощью `quick-xml`.
- [ ] Разработать менеджер плагинов (табличный вид в `egui`, установка/удаление плагинов).

## Этап 6: Кассовое оборудование (ККТ) и Порты 🖨️
- [ ] Перенести `VComCaster` (работа с COM-портами, настройка `com0com`).
- [ ] Реализовать логику установки и настройки драйверов ККТ (Атол, Штрих).
- [ ] Настроить сетевые маршруты (если используются для ККТ / Ethernet).

## Этап 7: Релиз и Полировка ✨
- [ ] Добавить всплывающие уведомления (Toasts) для успешных операций и ошибок.
- [ ] Написать модульные тесты для бизнес-логики Backend'а.
- [ ] Настроить GitHub Actions (CI/CD) для автоматической сборки Release `.exe` под Windows MSVC.
- [ ] Финальное тестирование на кассовом сенсорном терминале.
